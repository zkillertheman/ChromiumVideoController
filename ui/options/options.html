            The speed controls are not showing up for local videos or Incognito mode?
          

          

            To enable playback of local media (e.g. File > Open File) or Incognito mode, you need
            to manually grant additional permissions to the extension.
          

          
    
            
  • In a new tab, navigate to chrome://extensions
    • 
            
  • 
              Find "Video Speed Controller" extension in the list and enable "Allow access to file
              URLs" and/or "Allow in Incognito".
            
    • 
          


          
How do I disable the extension on a specific site?

          

            Go to Advanced > Site rules and add a rule with the site's domain
            (e.g. example.com). Check the Disable checkbox. The
            extension will not inject on that site. You can also use regex patterns like
            /\.edu$/i to match multiple domains.
          


          
A site's keyboard shortcuts conflict with mine. What can I do?

          
Three options:

          
    
            
  • 
              Add a modifier: In Settings > Shortcuts, add Shift,
              Ctrl, or Alt to your shortcut keys to create chord
              combinations the site won't intercept.
            
    • 
            
  • 
              Rebind to unused keys: Change the default key to one the site doesn't
              use.
            
    • 
            
  • 
              Enable exclusive mode: Turn on
              Exclusive keyboard shortcuts in Preferences to prevent sites from also
              handling your VSC shortcut keys.
            
    • 
          


          

            The controller overlaps with site controls. How do I reposition it?
          

          

            Go to Advanced > Controller CSS. Add a domain rule targeting
            vsc-controller using the --vsc-domain CSS variable. Example:
          

          

            :root[style*='--vsc-domain: "example.com"'] vsc-controller { position: relative; top:
              60px; }
          

          

            You can also drag the controller to a new position on any page. See the existing rules
            for YouTube, Netflix, etc. as reference.
          


          
The controller is not appearing on a site. Why?

          
Common causes:

          
    
            
  • 
              The site is in your Site rules with Disable checked — remove or
              uncheck it.
            
    • 
            
  • 
              The site uses Flash or a non-HTML5 player — the extension only works with native
              <video> and <audio> elements.
            
    • 
            
  • 
              Hide controller by default is enabled — press your display toggle
              shortcut (default: V) to show it.
            
    • 
          


          
Speed resets when I navigate on YouTube or similar single-page apps?

          

            YouTube uses SPA navigation — the page doesn't fully reload, but the player may
            reinitialize. This build always starts fresh media at 1x, so if a site
            replaces the underlying player you will need to set your speed again.
          


          
Does this work with audio elements, not just video?

          

            Yes. When Audio support is enabled in Preferences (on by default), the
            controller appears on <audio> elements too. All speed controls and
            keyboard shortcuts work identically for audio.
